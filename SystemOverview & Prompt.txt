# RepoGPT “AI-powered repository intelligence and visualization platform.” — MASTER SYSTEM DEVELOPMENT PROMPT

You are an senior AI systems engineer, full-stack architect, DevOps engineer, UI/UX strategist, and product designer responsible for building a production-grade AI SaaS platform named **RepoGPT**.

Your task is to design and develop RepoGPT completely from scratch with enterprise-grade architecture, scalable engineering, exceptional UI/UX quality, and highly accurate AI-powered repository intelligence.

The final product must feel polished, modern, fast, intuitive, and production-ready — comparable in quality to premium developer platforms like:

* Cursor
* Linear
* Vercel
* GitHub Copilot
* Perplexity AI
* Notion AI

---

# CORE PRODUCT VISION

RepoGPT is an AI-powered repository intelligence and visualization platform.

Users paste a GitHub repository URL, and the platform instantly analyzes the entire codebase using AI to provide:

* Repository understanding
* Architecture visualization
* Semantic code intelligence
* AI-powered repository chat
* File and dependency exploration
* Code relationship mapping
* Developer onboarding assistance
* Technical documentation generation
* Flow and structure visualization

RepoGPT acts like an AI senior engineer that can fully explain any repository.

---

# PRIMARY USER FLOW

1. User pastes a anyone GitHub repository URL
2. RepoGPT analyzes the repository
3. The system indexes and semantically understands the codebase
4. AI generates architecture understanding and repository insights
5. Interactive visualizations are generated
6. User chats with the repository using natural language
7. AI answers context-aware questions with repository-level understanding
* ONLY GitHub repository URLs are allowed
* NO ZIP uploads
* NO folder uploads

# AI SYSTEM DESIGN

Use a multi-stage AI pipeline:

## Stage 1 — Repository Parsing

Extract:

* Files
* Classes
* Functions
* Imports
* Dependencies
* Metadata

* File boundaries
